# EECS 498-016: Summer Preparation Plan

> Foundational skills to hit the ground running in Fall 2026

## Overview

The course moves fast from Day 1. These materials cover the **prerequisite skills** — the things we assume you already know when class starts. Spend the summer getting sharp on these fundamentals so you can focus on the hard stuff when it counts.

**Time commitment:** ~2-4 hours/week over 8 weeks, self-paced
**Not graded. Not required. Strongly recommended.**

## The Four Tracks

### Track 1: Python & TypeScript Fluency (Weeks 1-2)
Projects use both Python and TypeScript. We assume you can write clean, well-structured code in both on Day 1. This track covers Python 3.11+ (type hints, dataclasses, async/await, pathlib, httpx, pytest) and TypeScript (types, interfaces, generics, async/await, Node.js basics, testing). You'll build the same exercises in both languages to internalize the patterns.

### Track 2: Terminal & Git Mastery (Weeks 3-4)
You'll live in the terminal all semester. This track takes you beyond EECS 201 basics into real productivity: piping and text processing, environment variables, process management, SSH key setup, and scripting. On the Git side: branching strategies, interactive rebase, merge conflict resolution, `git log/diff/blame`, the GitHub CLI (`gh`), and fork/upstream workflows. The goal is muscle memory.

### Track 3: Building & Deploying Software (Weeks 5-6)
Writing code is only half the job — you also need to build, package, and ship it. This track covers project structure, build tools (Make, npm scripts, Python build tools), Docker basics (Dockerfile, docker-compose, containers, volumes), CI/CD with GitHub Actions, and deployment concepts. You'll Dockerize a real app, write a CI pipeline, and set up a proper Makefile.

### Track 4: HTTP, APIs & System Design (Weeks 7-8)
Modern software is built from components that communicate over networks. This track covers HTTP fundamentals (methods, status codes, headers), REST API design, building APIs in both Python (FastAPI) and TypeScript (Express), calling APIs with authentication, and how multi-process systems communicate (stdin/stdout, files, HTTP, sockets). You'll build a REST API, write a client for it, and chain two services together.

## How to Use This

- **Already strong in Python and TypeScript?** Skip Track 1, start at Track 2.
- **Comfortable in the terminal?** Skip Track 2.
- **Already Dockerize stuff and run CI?** Skim Track 3, focus on Track 4.
- **Coming from EECS 281 with solid C++?** Prioritize Tracks 1 and 3.

Each track has:
- A skills checklist (self-assess before diving in)
- Curated resources (not a textbook — just the good stuff)
- Practice exercises (build real things, not toy examples)
- A self-check (can you do these without looking anything up?)

## Prerequisites Refresher

This plan assumes you've completed **EECS 281** and **EECS 201** (or equivalent). If you're rusty on:
- **Data structures & algorithms** → Review EECS 281 materials, especially graphs and trees
- **Linux/CLI basics** → Review EECS 201 materials or [The Missing Semester](https://missing.csail.mit.edu/)
